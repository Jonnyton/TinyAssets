## Why

Nine shipped behavior groups remain materially absent from canonical OpenSpec even though they have no collision with an active implementation change. Until they are backfilled, `openspec/specs/` is truthful but incomplete as an as-built inventory.

## What Changes

- Specify current community-patch PR creation, loop health, nested Patch Packet extraction, and completed-run reuse.
- Specify cumulative multi-shot ASP validation, the installed desktop entrypoint, coordination drift/JSON diagnostics, the frozen authority-resolver contract, domain-owned branch/episodic registries and ScenePacket emission, provider bridge retries, Goal compatibility aliases, and wiki maintenance actions.
- Add an `external-effect-adapters` capability for the shipped GitHub PR/merge, Twitter, wiki-writeback, and Windows desktop sinks plus completion-path dispatch, evidence, and quarantine behavior. Shared authority, consent, and receipt semantics remain owned by `external-effect-receipts`.
- Record only current behavior and its current limitations; make focused test assertions and coordination-script output portable without changing product-runtime behavior.

## Capabilities

### New Capabilities

- `external-effect-adapters`: shipped completion-path sink dispatch, adapter-specific use of shared gates, run-snapshot evidence, forged-evidence quarantine, and current partial-write/finalization limitations.

### Modified Capabilities

- `community-patch-loop`: add the shipped auto-ship PR, health, nested-packet, and completed-run contracts.
- `constraint-evaluation`: add cumulative incremental multi-shot grounding behavior.
- `desktop-host-runtime`: add the canonical installed GUI entrypoint and distinguish it from the legacy source launcher.
- `development-coordination-runtime`: add required-artifact/skill-mirror drift checks, automation-facing JSON modes, and the shipped authority-resolver v1 decision contract.
- `domain-plugin-runtime`: add domain-owned Branch-slug and episodic-coordinate registries plus the shipped fantasy ScenePacket shape/emission path.
- `provider-routing`: add the three-attempt exhaustion retry and fallback contract.
- `shared-goals-and-convergence`: add the four shipped ChatGPT compatibility aliases.
- `wiki-commons`: add cosign, guarded delete, consolidation, lint, and project-sync semantics.

## Impact

This is an as-built specification reconciliation with focused test maintenance and one coordination-script newline portability fix. It updates nine canonical capability owners using evidence from the existing runtime and tests; it does not change product runtime, storage, public MCP behavior, or deployment state.
