## KEY ENTRY POINTS

### PRIMARY ORCHESTRATOR
workflow/memory/manager.py :: MemoryManager.assemble_context(phase, state)
  → Returns ContextBundle for ORIENT|PLAN|DRAFT|EVALUATE phases
  → Budget: 15K tokens
  → Sub-coordinators: promotion_gates, reflexion_engine

### QUERY LAYER
workflow/retrieval/router.py :: RetrievalRouter.retrieve(query, phase, scope)
  → Routes via HippoRAG (entity), RAPTOR (thematic), LanceDB (tone/similarity)
  → Decomposition: LLM routes compound queries to backends

### KNOWLEDGE INGESTION
workflow/ingestion/core.py :: ingest_file(file_path, dest_dir)
  → Detects file type, routes to extraction pipeline
  → Large files trigger synthesize_source signal

workflow/ingestion/indexer.py :: ingest_and_index(prose, universe_id, kg, scope)
  → Extract entities → build KG rows → insert facts

### FACT LIFECYCLE
workflow/memory/promotion.py :: PromotionGates.evaluate(facts, scene_refs)
  → Promotion: facts with sufficient evidence → canon
  → Decay: persistent violations → ASP rule candidates

## CRITICAL DESIGN PATTERNS

### SCOPING THREADING (Stage 2b)
All tables have 4-tier columns: (universe_id, goal_id, branch_id, user_id)
  - episodic_facts, scene_summaries (memory/episodic.py)
  - entities, edges, facts, communities (knowledge/knowledge_graph.py)
  
Stage 2c: WORKFLOW_TIERED_SCOPE flag flips to enable read-side filtering
  - Pending implementation (design complete, tests exist)

### SINGLETON PATTERNS WITH CROSS-UNIVERSE GUARDS
get_db(path) :: LanceDB connection
  - Requires explicit path (no cwd-relative defaults)
  - Thread-safe via _lock
  - reset_db() for testing only

KnowledgeGraph(db_path) :: SQLite connection
  - Requires explicit path (same guard as vectors)
  - Thread-safe via _lock
  - Design note: prevents two universes sharing relative path

### GRACEFUL FALLBACKS
ArchivalMemory._try_load_kg(), _try_load_asp()
  - Returns empty results when backends unavailable
  - Allows memory system to bootstrap standalone
  - No hard dependencies on KG or Clingo

### PHASE-AWARE CONTEXT ASSEMBLY
MemoryManager._assemble_{orient,plan,draft,evaluate}()
  - Calls phase-specific sub-builders
  - Budget tracking: trim_to_budget() if >15K tokens
  - Debug mode: WORKFLOW_DEBUG_CONTEXT=1

## DEPENDENCY FLOW (simplified)

Memory
  ├─ depends on Knowledge (ArchivalMemory delegates to KG)
  ├─ depends on Learning (LearningSystem integrated in manager.py)
  ├─ depends on Ingestion (ProgressiveIngestor coordinates canon file flow)
  └─ depends on Storage (episodic SQLite path)

Retrieval
  ├─ depends on Knowledge (Router dispatches to HippoRAG/RAPTOR)
  ├─ depends on Memory (MemoryScope threading)
  └─ depends on Storage (VectorStore path)

Knowledge
  ├─ depends on Constraints (graceful fallback to ASP)
  ├─ depends on Storage (SQLite path)
  └─ depends on Ingestion (entity_extraction called by indexer)

Learning
  ├─ depends on Memory (promotion_gates, style_observations)
  └─ depends on Knowledge (entity relationships for consistency checks)

Ingestion
  ├─ depends on Knowledge (entity extraction, indexer → KG)
  └─ depends on Storage (manifest.json, canon/ paths)

## ENVIRONMENT & RUNTIME SETUP

Variables:
  WORKFLOW_DEBUG_CONTEXT=1      → memory/manager.py detailed logging
  WORKFLOW_TIERED_SCOPE=off     → retrieval/router.py flag (Stage 2b → 2c)

Paths:
  Episodic DB:  passed as db_path to MemoryManager (default: ":memory:")
  Knowledge DB: passed as db_path to KnowledgeGraph
  Vector store: LanceDB directory (must be explicit, not cwd-relative)
  Canon:        universe_home/canon/ (ingest/core.py)
  Manifest:     universe_home/canon/.manifest.json

Singletons:
  MemoryManager:   one per universe (graph-core nodes)
  KnowledgeGraph:  one per universe (created on demand)
  VectorStore:     one per universe (created on demand)
  LanceDB.get_db: singleton connection (thread-safe lock)
