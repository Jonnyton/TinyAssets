# Discovery Flows — Six Doors, Walked (2026-07-09)

**Status:** Binding GTM design note. Each persona: where they hear of us -> first click -> first-session value -> break risks + fixes. Companion to USER-PATH (which covers post-arrival); this covers ARRIVAL.

## Universal law: the anonymous-first gradient
Browse/read the commons with ZERO account (MCP read paths + public artifact pages). Account appears only when something persists. Wallet/funding appears only at first market spend (seam-1, fiat-first). Any flow demanding signup before value is wrong-shaped.

## 1. Power user (OpenClaw/Hermes/Obsidian) — demand + designs
Hears: ClawHub listing, r/LocalLLaMA, HN. First click: ONE command from inside their existing agent — connect, browse archetypes, clone into a universe. First value (<5 min): a shaped, persistent, portable cloud brain; their setup already mirrored in the archetype gallery ("it speaks my shape").
**Requires:** six seeded archetypes · a published ClawHub skill (the listing IS the marketing).

## 2. Agent-routed ("I want to build something," to any chatbot)
Works only if agents can find + read us. **Requires:** llms.txt + AGENTS.md on tinyassets.io · MCP directory listings · **every commons artifact gets a public crawlable page** (the commons IS the content marketing: humans via search, agents via clean markdown, both ending at "open in your chatbot"). Closed vendor directories assumed late/hostile (OPERATING-NOTES §5); open web + OpenClaw route around.

## 3. PC gamer/modder — supply: compute
Hears: earnings screenshots, builder-community content ("my 4090 bought my coffee"). First click: **the earnings calculator** (pick GPU -> est. monthly at current spot) — MISSING ARTIFACT, now required; needs the live price index (Track E W3b) behind it. First value: daemon install -> benchmark -> first PAID job within days.
**HARD SEQUENCING RULE:** supply doors open ONLY after utilization is real. Premature gamer marketing -> first cohort earns cents -> "scam" posts in the exact community we need. The demand floor for first sellers is honest platform-native work: every universe's gardener + seeded Dataset-Forge/archetype bounties are genuine batch jobs. Early sellers get paid maintaining users' brains; their receipts are the next wave's marketing.

## 4. Maker/builder — supply: fabrication
Hears: (a) seeded fab bounties ("get paid to print" — founder incentive budget doubling as demo material: carrier enclosures, cookbook shells), (b) later, the attribution-claim email ("your Printables design earned $X — claim it") once remix volume exists. Claim flow must be one verification step (proof of origin-account control). Slowest door; stickiest supply.

## 5. SMB ops — the fourth door (monetizes first, marketed last)
Hears: search ("automate weekly payables"), word of mouth. **VOCABULARY RULE:** this door NEVER sees "universe/commons/token" — its landing lens speaks its language ("Payables runs itself every Thursday. Connect Gmail + your accounting tool."), onboards directly into the payables archetype, bills fiat, hides rails entirely. Per-door landing pages are LENSES (commons-architecture §2) with door-native language. Sequence after core reliability — zero tolerance for a missed Thursday.

## 6. Vibe coder with an idea
Hears: the demo (cookbook) going wide, launch content. Flow = the demo runbook path; first value = harness in seconds + a standing goal that lands a win in week one.

## Distribution-artifact checklist (pre-launch, joins the seed list)
[ ] Six archetypes (existing) [ ] ClawHub connector skill [ ] llms.txt + AGENTS.md on tinyassets.io [ ] MCP directory listings [ ] Public crawlable page per commons artifact [ ] Earnings calculator (gated on price index) [ ] Per-door landing lenses (power-user / gamer / maker / SMB / builder) [ ] Attribution-claim flow (one-step origin verification) [ ] Seeded bounty board (founder-funded, real gaps)

## Sequencing summary
Demand doors (1, 2, 6) at launch -> platform-native work + bounties create the demand floor -> supply doors (3, then 4) once utilization is screenshot-worthy -> SMB lens (5) once Thursday never misses.
