## BRAIN OPERATION TYPES (Taxonomy)

ingest   — write new knowledge / state
  wiki: write, promote, ingest, supersede, consolidate, file_bug, cosign_bug, sync_projects
  universe: submit_request, add_canon, set_premise
  extensions: project_memory_set

query    — read existing knowledge / state
  wiki: read, search, list, lint
  universe: read_output, read_premise, read_canon, query_world
  extensions: project_memory_get, project_memory_list

digest   — refine / clean / deduplicate
  wiki: promote, consolidate, supersede, lint (validation)
  
lint     — validate integrity
  wiki: lint (orphans, broken links, stale, etc.)

export   — push to external system
  scripts/wiki_bug_sync.py → GitHub Issues

## SCHEMA MATRIX (What Each Brain Surface Reads/Writes)

Wiki Pages/Drafts
  ├─ pages/<category>/<slug>.md
  │  ├─ Frontmatter (YAML)
  │  │  ├─ title: str [required for promote]
  │  │  ├─ type: str [required for promote]
  │  │  ├─ sources: str | path: str [required unless type=project]
  │  │  ├─ confidence: enum(high, low, superseded)
  │  │  ├─ updated: ISO-8601 [auto-set on promote]
  │  │  └─ superseded_by: [[wikilink]] [if confidence=superseded]
  │  └─ Body (Markdown with [[wikilinks]])
  │
  ├─ drafts/<category>/<slug>.md (same structure, not promoted)
  │
  └─ pages/bugs/<BUG-NNN>-<slug>.md [special]
     └─ Frontmatter (auto-generated by file_bug)
        ├─ bug_id: BUG-NNN | FEAT-NNN | DESIGN-NNN
        ├─ component: str
        ├─ severity: enum(critical, major, minor, cosmetic)
        ├─ kind: enum(bug, feature, design)
        ├─ first_seen_date: ISO-8601
        ├─ tags: list[str]
        ├─ status: enum(open, in-progress, closed)
        ├─ investigation: {status, method, findings}
        └─ trigger: {request_id, receipt}

Project Memory (per-universe, per-project)
  └─ <universe>/.project_memory/<project_id>/<key>.json
     ├─ value: JSON
     ├─ version: int
     └─ updated_at: ISO-8601

Universe Requests (per-universe, append-only)
  └─ <universe>/requests/<request_id>.json
     ├─ request_type: enum(scene_direction, task, query, ...)
     ├─ body: str | JSON
     ├─ provenance: str
     ├─ created_at: ISO-8601
     └─ status: enum(pending, in-progress, completed, failed)

Universe Canon (per-universe, reference library)
  └─ <universe>/canon/<filename>
     (any prose, code, or data)

Ledger (audit trail, per-universe)
  └─ <universe>/ledger.json
     └─ [{action, target, summary, created_at}, ...]

## ROUTING DECISION TREE (for chatbots)

"I want to save/remember X"
├─ Is it knowledge/reference/how-to? → wiki.write → wiki.promote
├─ Is it project-specific state? → extensions project_memory_set
├─ Is it universe-scoped direction? → universe submit_request
└─ Is it a bug/feature/design defect? → wiki.file_bug

"I want to find/search X"
├─ Is it reference knowledge? → wiki.search
├─ Is it a bug? → wiki.search + wiki.lint (orphans?)
├─ Is it workspace state? → universe.read_output or query_world (stub)
└─ Is it project state? → extensions project_memory_get

"I want to clean up / improve"
├─ Are there duplicate drafts? → wiki.consolidate (dry_run first!)
├─ Are there broken wiki links? → wiki.lint
├─ Is content stale/obsolete? → wiki.supersede
└─ Is a page unhealthy? → wiki.promote (skip_lint=false)
